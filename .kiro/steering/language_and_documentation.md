# Convenciones de Idioma y Documentación

Este documento define las reglas lingüísticas para el desarrollo de la aplicación de rutinas de gimnasio. Kiro debe seguir estas directrices en todas las interacciones de generación y edición de código.

## 1. Comentarios de Código
- **Idioma:** Todos los comentarios dentro de los archivos de código (.dart, .java, etc.) deben escribirse exclusivamente en **español**.
- **Estilo:** Usa un tono profesional y técnico.
- **Traducción:** Si encuentras comentarios existentes en inglés al refactorizar, tradúcelos al español manteniendo el sentido técnico original.

## 2. Documentación de Archivos
- **README y Guías:** Todos los archivos de documentación (README.md, manuales técnicos) deben estar en español.
- **Documentación de API:** Las descripciones de métodos y clases (DartDoc) deben redactarse en español.

## 3. Nombramiento de Variables y Clases
- **Idioma:** Para mantener la compatibilidad con los estándares de la industria y las librerías de Flutter/Spring Boot, los nombres de clases, funciones, variables y archivos deben permanecer en **inglés** (ej. `WorkoutRepository`, `getExerciseById`). [3]

## 4. Interfaz de Usuario (UI)
- **Strings:** Todo el texto visible para el usuario final en la aplicación Flutter debe estar en **inglés**.
